# Security Policy

## Supported Versions

Security fixes are currently targeted at the active `1.x` line.

| Version | Supported |
| --- | --- |
| `1.x` | Yes |
| `< 1.0` or unpublished pre-release work | No formal support window |

## Reporting A Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, report them privately to:

- `jovanie.daclizon@gmail.com`

Include as much of the following as possible:

- package version
- PHP version
- Laravel version
- affected classes or methods
- reproduction steps
- proof of concept or minimal payload
- impact assessment
- any suggested remediation

## What To Expect

Best-effort process:

1. acknowledgement of receipt
2. validation and impact review
3. coordinated fix and release planning
4. disclosure guidance once a fix is available

## Disclosure Guidance

Please avoid:

- public issue reports before a fix is available
- publishing proof-of-concept exploits before maintainers have time to respond

If you are unsure whether something is a security issue or a correctness bug, report it privately first.
