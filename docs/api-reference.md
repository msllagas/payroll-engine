# API Reference

This document is a high-level reference for the package's main public classes, DTOs, and extension contracts.

## Core Entry Points

### `QuillBytes\PayrollEngine\PayrollEngine`

Responsibility:

- primary orchestration service for payroll computation and output generation

Main methods:

| Method | Input | Output | Use when |
| --- | --- | --- | --- |
| `compute($company, $employee, $input)` | one company, one employee, one payroll input | `PayrollResult` | preview or finalize payroll for one employee |
| `run($company, $period, $items)` | one company, one payroll period, many employees | `PayrollRun` | compute a payroll cutoff |
| `payslip(PayrollResult $result)` | one payroll result | array payload | generate one payslip-ready structure |
| `payrollRegister(array $results)` | many payroll results | array of rows | generate register/export data |
| `allocationSummary(array $results, string $dimension)` | many payroll results plus dimension | grouped rows | summarize by project, branch, department, cost center, vessel |
| `retroAdjustmentInput($original, $recomputed, $releasePeriod)` | two results plus release period | `PayrollInput` | release difference-only retro payroll |
| `generatePayrollFiles(PayrollRun $run)` | processed payroll run | array of rows | generate run-scoped register/export data |
| `generatePayslips(PayrollRun $run, ?CarbonImmutable $generatedAt = null)` | processed/releasable payroll run | array of payslip payloads | generate run-scoped payslips |

Extension notes:

- resolves client-specific strategies internally
- runs edge-case policies before and after the workflow
- enriches results with audit metadata

### `QuillBytes\PayrollEngine\PayrollEngineFacade`

Responsibility:

- Laravel facade wrapper around `PayrollEngine`

Use when:

- you prefer `\PayrollEngine::compute(...)` style access in Laravel

### `QuillBytes\PayrollEngine\PayrollEngineServiceProvider`

Responsibility:

- integrates the package into Laravel's container
- publishes config
- binds the main engine singleton

Use when:

- understanding how the package is wired into Laravel
- manually registering the package when discovery is disabled

## Core Data Objects

### `PayrollInput`

Responsibility:

- normalized payroll input for one employee and one payroll period

Contains:

- period
- overtime entries
- variable earning entries
- adjustments
- manual deductions
- loan deductions
- bonus
- used annual bonus shield
- optional projected annual taxable income
- metadata

Use when:

- building or transforming payroll input before calling the engine

### `PayrollResult`

Responsibility:

- result object for one employee payroll computation

Contains:

- normalized company, employee, and period
- rates
- earnings and deductions
- employee and employer contributions
- separate payouts
- gross pay, taxable income, net pay, and take-home pay
- bonus tax withheld
- issues
- audit metadata

Use when:

- rendering payslips
- persisting payroll history
- reviewing audit data

### `PayrollRun`

Responsibility:

- batch payroll result for one company and one period

Contains:

- company
- period
- `results`
- lifecycle status
- audit timestamps and state transitions

Key lifecycle helpers:

- `prepare()`
- `approve()`
- `process()`
- `reopen()`
- `release()`
- `assertEditable()`
- `assertCanGeneratePayrollFiles()`
- `assertCanGeneratePayslips()`

Use when:

- running payroll in batch
- enforcing release workflow rules

### `RateSnapshot`

Responsibility:

- normalized rate data used by the payroll workflow

Contains:

- `monthlyBasicSalary`
- `scheduledBasicPay`
- `dailyRate`
- `hourlyRate`
- `fixedPerDayApplied`

Use when:

- custom strategies need the resolved rates
- reports need to display rate details

## Report Builders Exposed Through `PayrollEngine`

### Payslip payload

Generated by:

- `PayrollEngine::payslip()`
- `PayrollEngine::generatePayslips()`

Use when:

- rendering PDF or Blade payslips
- returning an employee-facing payroll view payload

### Payroll register payload

Generated by:

- `PayrollEngine::payrollRegister()`
- `PayrollEngine::generatePayrollFiles()`

Use when:

- generating CSV or finance export rows

### Allocation summary

Generated by:

- `PayrollEngine::allocationSummary()`

Use when:

- grouping payroll totals by cost center, project, branch, department, or vessel

## Strategy Contracts

### `PayrollWorkflow`

Responsibility:

- full payroll result construction

Use when:

- a tenant needs to replace the entire computation sequence

### `RateCalculator`

Responsibility:

- scheduled basic pay, daily rate, and hourly rate resolution

Use when:

- a tenant needs custom rate derivation while preserving the default workflow

### `OvertimeCalculator`

Responsibility:

- compute overtime earning lines

Use when:

- OT rules or multipliers differ by tenant

### `VariableEarningCalculator`

Responsibility:

- convert variable earning entries into payroll earning lines

Use when:

- commissions, incentives, or performance earnings need custom logic

### `WithholdingTaxCalculator`

Responsibility:

- regular withholding tax and bonus tax computation

Use when:

- a tenant needs a different annualization model or tax table

### `PagIbigContributionCalculator`

Responsibility:

- Pag-IBIG contribution and related deduction handling

Use when:

- Pag-IBIG rules differ from the package default

### `PayrollEdgeCasePolicy`

Responsibility:

- prepare or finalize contextual payroll exceptions

Methods:

- `prepare()`
- `finalize()`

Use when:

- you need pre- or post-compute special handling such as attendance rules, net-pay protection, or custom warnings

## Supporting Policy Classes

### `ClientPolicyRegistry`

Responsibility:

- resolve client-specific preset defaults before published config overrides are applied

### `PayrollEdgeCasePolicyPipeline`

Responsibility:

- orchestrate ordered prepare/finalize policy execution around payroll computation
